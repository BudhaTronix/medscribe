WHAT IT DOES An end-to-end local pipeline for clinical documentation. Spoken clinical dictation is transcribed with faster-whisper, turned into a schema-validated structured note by a local LLM through Pydantic, and grounded question answering runs over a small bilingual German and English medical reference corpus using Qdrant vector search. Everything runs locally with no paid APIs. All medical content is synthetic and generated by you.

PYDANTIC SCHEMA (implement verbatim; add validators only) from typing import Literal from pydantic import BaseModel, Field class Medication(BaseModel): name: str dose: str | None = None frequency: str | None = None class ClinicalNote(BaseModel): language: Literal["de", "en"] chief_complaint: str = Field(min_length=3) history_of_present_illness: str medications: list[Medication] = [] allergies: list[str] = [] assessment: str plan: list[str] = Field(min_length=1) red_flags: list[str] = [] uncertainty_notes: str = ""

BEHAVIOUR Chunker: fixed-size token chunks with overlap as default (approximate tokens as whitespace words times 0.75, document this), plus a paragraph-aware mode behind a flag. Unit test both. Ingestion embeds chunks and upserts to Qdrant with payload {doc_id, title, chunk_index, text, language}. Retrieval: dense top-k with a score threshold. Structured extraction (llm/extraction.py): build a prompt containing the transcript and the schema from ClinicalNote.model_json_schema(); request JSON only; parse with ClinicalNote.model_validate_json. On ValidationError retry up to MAX_VALIDATION_RETRIES, each time appending the validation error text with an instruction to fix and return only corrected JSON. After the final failure return a typed failure object with the raw output and the error list, never raise to the caller. Log which attempt succeeded. Grounded Q&A (llm/rag.py): embed the question, search Qdrant top-k; if the best score is below SCORE_THRESHOLD return a refusal answer stating the corpus lacks the information, with zero citations and no LLM call. Otherwise build a context block of retrieved chunks with ids and titles, instruct the model to answer only from context and to say when context is insufficient, and return answer, citations (doc_id, title, chunk_index, score), and timings.

API ENDPOINTS GET /health/live (always 200 while serving). GET /health/ready (200 only when Qdrant is reachable, the collection exists with points, and the LLM endpoint answers a model-list call; otherwise 503 with a component status body for qdrant, llm, embedding_model_loaded; load the embedding model at startup and report ready only after). POST /transcribe (multipart audio; returns text, segments with timestamps, detected language, real-time factor, timings). POST /notes/structure (accepts transcript text or an audio file; returns ClinicalNote or the typed failure object plus timings). POST /ask (json question; supports a generate=false query flag that stops after retrieval and returns citations only, for load testing; returns answer, citations, timings). POST /ingest (runs the corpus pipeline; returns counts).
